What is this system?

This system is a bridge between Mixxx and Rekordbox, designed to facilitate the transfer of playlists and musical organization between these two platforms.

‚ö†Ô∏è Important: This program does NOT replace Rekordbox. It is a complementary tool that allows you to automatically import your organizational structures from Mixxx to Rekordbox through an XML file.

How does Rekordbox work with USB drives?

Rekordbox is Pioneer DJ's music management software that allows you to prepare and organize your music for use on compatible equipment such as the CDJ-3000.

USB export process:

  1. Playlist export: Rekordbox exports the playlists you have loaded to a USB drive.
  2. Audio file copy: During export, Rekordbox copies the audio files (MP3, FLAC, etc.) to the USB drive.
  3. Metadata generation: Rekordbox analyzes each track and generates:
    • Visual waveforms
    • Cue and loop information
    • BPM and tempo analysis
    • Musical key information
    • Other metadata needed for compatible players
  4. Proprietary format: All this information is stored in a proprietary format that only Rekordbox can generate and that Pioneer equipment can read.
üí° Note: For Rekordbox to analyze and generate all this information, you need to have the music previously loaded and analyzed in Rekordbox, along with manually created playlists.

What problem does this program solve?

If you already have your music organized in Mixxx with playlists and/or crates, creating a Rekordbox-compatible USB drive traditionally requires:

  1. Having the music accessible in Rekordbox
  2. Manually creating all the playlists and crates you have in Mixxx as playlists in Rekordbox
  3. Exporting everything to USB from Rekordbox

This manual process can be very tedious if you have many playlists and crates organized in Mixxx.

The solution:

This program generates an XML file that Rekordbox can read directly. By importing this XML into Rekordbox, all the playlists you have in Mixxx will be automatically created, without needing to create them manually.

System requirements

If you use Linux:

You will need access to Rekordbox, which is only available for Windows and macOS. You have two options:

  • Physical Windows machine: A Windows computer where you can install Rekordbox
  • Windows virtual machine: A virtual machine running Windows on your Linux system
‚ö†Ô∏è Important: In both cases, Rekordbox must be able to access the music files. This means you will need to:
  • Copy the music to a location accessible to Rekordbox
  • Make sure Rekordbox can find and analyze the files

The file path problem

One of the challenges when working between Linux and Windows is that file paths have different formats. However, it is important to maintain the same directory structure for the mapping to work correctly.

‚ö†Ô∏è Important: The subdirectory structure must be identical in both systems. Only the first part of the path (the prefix/base) changes. For example, if in Linux you have /home/user/Music/electronic/house/track1.mp3, in Windows you should have D:\electronic\house\track1.mp3 (maintaining the electronic/house/ structure).

Solution: Path mapping

This program allows you to specify the Original Base Path (the initial part of the path in Linux) and the Destination Base Path (the equivalent initial part in Windows). You only need to indicate the difference in the first part of the paths; the program will automatically replace this base part while keeping the rest of the directory structure intact.

Practical examples:

Example 1 - Simple path:
Original Base Path (Linux): /home/user/Music/electronic
Destination Base Path (Windows): D:\electronic

File in Linux: /home/user/Music/electronic/house/track1.mp3
Will be converted to: D:\electronic\house\track1.mp3

Note: The house/ structure remains the same in both systems.
Example 2 - Path with Windows user:
Original Base Path (Linux): /home/user/Music
Destination Base Path (Windows): C:\Users\User\Music

File in Linux: /home/user/Music/electronic/techno/set1.mp3
Will be converted to: C:\Users\User\Music\electronic\techno\set1.mp3

Note: The electronic/techno/ structure remains the same.
Example 3 - External drive or network unit:
Original Base Path (Linux): /media/usb/Music
Destination Base Path (Windows): E:\Music

File in Linux: /media/usb/Music/playlists/party/track2.mp3
Will be converted to: E:\Music\playlists\party\track2.mp3

Note: The playlists/party/ structure remains the same.
üí° Tip: To correctly identify the base paths, take any file from your library and compare the paths:

In Linux: /home/user/Music/electronic/subfolder/file.mp3
In Windows: D:\electronic\subfolder\file.mp3

The common part that changes is: /home/user/Music ‚Üí D:\
Therefore, you would use:
‚Ä¢ Original Base Path: /home/user/Music
‚Ä¢ Destination Base Path: D:\

How to use this program?

Step 1: Prepare your Mixxx database

You need to locate the Mixxx database file. It is usually located at:

Linux: ~/.mixxx/mixxxdb.sqlite
Windows: %LOCALAPPDATA%\Mixxx\mixxxdb.sqlite
macOS: ~/Library/Application Support/Mixxx/mixxxdb.sqlite

Note: Make sure to close Mixxx before copying the file to avoid locking issues.

Step 2: Configure the paths

In the application, complete the following fields:

  • Original Base Path (Linux): The base path where you have your music in Linux. For example: /home/user/Music/electronic
  • Destination Base Path (Windows): The base path where the files will be located in Windows. For example: D:\electronic or /D:/electronic
üí° Tip: You only need to specify the common part of the path. The program will automatically replace this base part in all file paths.

Step 3: Select what to include

You can choose which elements to include in the generated XML:

  • Include playlists: Check this option if you want to include your Mixxx playlists
  • Include crates: Check this option if you want to include your Mixxx crates
‚ö†Ô∏è Important: You must select at least one of the two options (playlists or crates). The program will not allow generating an XML without any playlists, as it would not make sense to create an empty file for Rekordbox. Both options are checked by default, but you can uncheck one of them if you only need to import playlists or only crates.

Step 4: Generate the XML

  1. Select the mixxxdb.sqlite file from Mixxx
  2. Complete the path fields
  3. Select which elements to include (playlists and/or crates)
  4. Click "Process and Generate XML"
  5. The XML file will be automatically downloaded

Step 5: Import into Rekordbox

Once you have the generated XML file, you need to import it into Rekordbox. There are two main methods to do this:

‚ö†Ô∏è IMPORTANT WARNING:
  • When importing the XML file into Rekordbox, existing data in your Rekordbox library may be overwritten, including playlists, metadata, or configurations you already have.
  • It is strongly recommended to make a backup of your Rekordbox library before importing the XML. You can do this by exporting your current library from Rekordbox (File ‚Üí Export ‚Üí Export Collection) or by making a backup copy of Rekordbox files.
  • If you have doubts about how to proceed, try importing the XML first in a test installation of Rekordbox or in a different user profile.

Method 1: Import through File menu (Recommended)

  1. Open Rekordbox on Windows or macOS
  2. Go to the top menu and select File ‚Üí Import ‚Üí Playlist or File ‚Üí Import ‚Üí Import Playlist File
  3. In the dialog box that appears, navigate to the location where you saved the generated XML file (usually downloaded to your Downloads folder)
  4. Select the XML file (it should be named rekordbox_export.xml or similar)
  5. Click Open or Import
  6. Rekordbox will begin importing the playlists. You will see progress at the bottom of the window
  7. Once the import is complete, you will see all your playlists and crates in the left panel of Rekordbox, under the "Playlists" section

Method 2: Configure as external XML library

This method is useful if you want to keep the XML as a separate library that Rekordbox can read without directly importing:

  1. Open Rekordbox and go to File ‚Üí Preferences (or Rekordbox ‚Üí Preferences on macOS)
  2. In the preferences window, look for the Advanced or Database tab
  3. Find the "rekordbox xml" or "External Library" section
  4. Click the Browse or Select button next to "Library Path" or "XML Library Path"
  5. Navigate and select the generated XML file
  6. Click OK to save the changes
  7. In the left panel of Rekordbox, look for the "rekordbox xml" or "External Library" section
  8. Click the refresh icon (usually a circle with arrows) next to the library name to load the content
üí° Note: Method 1 is more direct and recommended for most users, as it imports the playlists directly into your main Rekordbox library. Method 2 is useful if you prefer to keep the playlists as a separate external library.
‚ö†Ô∏è Important:
  • Make sure the music files are in the location specified in the "Destination Base Path" before importing the XML into Rekordbox
  • Rekordbox needs to be able to access the files to analyze them and generate waveforms
  • If the files are not in the correct location, Rekordbox will show the playlists but the tracks will appear as "missing files"
  • After importing, let Rekordbox analyze all the files. This may take several minutes depending on how much music you have
  • Once Rekordbox finishes analyzing the files, you can export everything to your USB drive

Verification after import

After importing the XML, verify that everything was imported correctly:

  • Check that all your playlists appear in the left panel of Rekordbox
  • Open some playlists and verify that they contain the correct tracks
  • Verify that tracks do not appear as "missing files"
  • If you see missing files, verify that the paths in the XML match the actual paths of your files in Windows

Additional information

What information is transferred?

The generated XML includes:

  • Tracks: Information for all tracks in your library
  • Metadata: Artist, title, album, genre, BPM, musical key, etc.
  • Position Marks: Cues and hot cues you have configured in Mixxx
  • Playlists: All your playlists with their tracks
  • Crates: All your organizational crates
  • Colors: Color labels assigned to tracks
  • Ratings: Track ratings

Limitations

  • This program does NOT generate waveforms or analyze audio files. That must be done by Rekordbox after importing the XML.
  • Music files must be physically available at the specified location for Rekordbox to analyze them.
  • Some specific Mixxx features may not have a direct equivalent in Rekordbox and may be lost in the conversion.

Tips

  • Make a backup of your Mixxx database before processing it
  • Verify the paths before generating the XML to make sure they match your configuration
  • If you have many tracks, the process may take a few moments. Be patient and don't close the page while it's processing
  • After importing into Rekordbox, let it analyze all files before exporting to USB to ensure all information is available

Playlist nomenclature

To facilitate identification, the program adds prefixes to playlist names:

  • [P] at the beginning of the name indicates it is a Playlist
  • [C] at the beginning of the name indicates it is a Crate

For example, if you have a playlist called "House Mix" and a crate called "_intros", they will appear as:

[P]House Mix
[C]_intros

This will help you easily distinguish between playlists and crates when you see them in Rekordbox.
